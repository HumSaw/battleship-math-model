# Security policy

## Supported versions

Security fixes are applied to the current `main` branch and the latest release.

## Reporting a vulnerability

Do not open a public issue for a security vulnerability. Use GitHub’s private vulnerability reporting feature on the repository Security tab. Include affected versions, reproduction steps, impact, and any suggested mitigation.

The application performs analysis entirely in the browser and does not require accounts or store board data on an application server. Dependency, build-chain, browser-isolation, and deployment configuration vulnerabilities are still in scope.

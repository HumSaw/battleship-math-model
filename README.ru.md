Браузерный советник для «Морского боя», который оценивает каждую допустимую цель по текущему состоянию поля. Движок сочетает точный перебор расстановок флота, взвешенное Sequential Importance Sampling, двухходовый lookahead и ограниченный expectimax в малых эндшпилях. Интерфейс переведён на 10 языков с автоопределением языка браузера.

> Это инструмент вероятностной поддержки решений, а не гарантия победы. Рекомендация оптимальна только относительно модели, наблюдений, набора правил и доступного вычислительного бюджета.

## Языки интерфейса

Интерфейс полностью локализован на 10 языков: **English (английский), 中文 (китайский), Español (испанский), हिन्दी (хинди), العربية (арабский, RTL), Português (португальский), বাংলা (бенгальский), Русский, 日本語 (японский), Français (французский)**.

Язык определяется по браузеру при первом входе, переключается селектором в шапке, ручной выбор сохраняется локально. Арабский отображается в зеркальной RTL-раскладке.

## Возможности

- Локализованный интерфейс на 10 языках с автоопределением языка и поддержкой RTL
- Русские классические правила и международные правила (Hasbro)
- Тепловая карта вероятностей 10×10
- Точный перебор допустимых расстановок при малом пространстве состояний
- Взвешенное Монте-Карло (Sequential Importance Sampling) для больших позиций
- Двухходовый условный lookahead для близких кандидатов
- Ограниченный expectimax в счётных эндшпилях
- Режимы поиска, добивания, победы и обнаружение противоречий
- Анализ в Web Worker с отменой устаревших запросов
- История отмен, явный учёт потопленных кораблей, горячие клавиши
- Детерминированные тесты движка, пороги покрытия, lint, typecheck и CI

## Как это работает

Для наблюдений $$O$$ и допустимой конфигурации флота $$C$$ движок оценивает апостериорную занятость каждой неизвестной клетки:

$$P(\text{корабль в } x \mid O) = \frac{\sum_C w(C)\,\mathbf{1}[x \in C]}{\sum_C w(C)}$$

При большом пространстве состояний конфигурации сэмплируются последовательно и взвешиваются числом допустимых вариантов на каждом шаге размещения. Для близких кандидатов политика оценивает ожидаемое качество следующего выстрела:

$$Q(x) = q_x\left(1 + \max_y P(y \mid x\text{ попал})\right) + (1-q_x)\max_y P(y \mid x\text{ промах})$$

Ручные «шахматные» бонусы сознательно не используются: геометрия флота уже учтена в апостериорных вероятностях, а дополнительная эвристика может выбрать строго худшую цель.

Детали реализации и ограничения — в [заметках об алгоритме](docs/ALGORITHM.md) и [архитектуре](docs/ARCHITECTURE.md) (на английском).

## Быстрый старт

Требования: Node.js 22+ и pnpm 10+.

```bash
git clone https://github.com/HumSaw/battleship-math-model.git
cd battleship-math-model
pnpm install
pnpm dev
```

Откройте `http://localhost:3000`.

## Проверки качества

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm test:coverage
pnpm build
```

## Воспроизводимый симулятор

В репозитории есть сидированный многопоточный симулятор на равномерно сгенерированных допустимых флотах:

```bash
pnpm simulate -- \\
  --games 1000 --workers 4 --rules russian \\
  --samples 1200 --budget 30 --seed 42
```

Результаты симуляции чувствительны к числу сэмплов, бюджету на ход, скорости CPU, приору флота и набору правил. Не сравнивайте числа без совпадения всех параметров и commit SHA.
